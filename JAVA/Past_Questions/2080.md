## Question 1: Why do we need to synchronize the thread? Justify with an example. An array with an odd number of elements is said to be centered if all elements (except the middle one) are strictly greater than the value of the middle element. Note that only arrays with an odd number of elements have a middle element. Write a function that accepts an integer array and returns 1 if it is a centered array, otherwise it returns 0.


**Thread Synchronization:**
- When multiple threads access and modify shared resources (like variables, files, or databases) without proper synchronization, it can cause race conditions, where the outcome depends on the timing of the threads' execution.
- Synchronization ensures that only one thread can access the shared resource at a time, thus maintaining data integrity and preventing conflicts.

**Need for Synchronization**
- Synchronization is required to prevent data inconsistency when multiple threads access shared data.
- It avoids race conditions where the output depends on the unpredictable execution order of threads.
- It ensures that only one thread executes a critical section at a time.
- It maintains data integrity in multi-threaded applications.
- It is important when threads perform read and write operations on shared variables.

**Example of Thread Synchronization:**
```java
class Counter {
    private int count = 0;

    // Synchronized method to increment the count
    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}
class MyThread1 extends Thread {
    private Counter counter;

    public MyThread1(Counter counter) {
        this.counter = counter;
    }

    @Override
    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }
}
class MyThread2 extends Thread {
    private Counter counter;

    public MyThread2(Counter counter) {
        this.counter = counter;
    }

    @Override
    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }
}
public class ThreadSynchronizationExample {
    public static void main(String[] args){
        Counter counter = new Counter();
        MyThread1 thread1 = new MyThread1(counter);
        MyThread2 thread2 = new MyThread2(counter);

        thread1.start();
        thread2.start();

        thread1.join();
        thread2.join();

        System.out.println("Final count: " + counter.getCount());
    }
}
```

- In this example, we have a `Counter` class with a synchronized method `increment()` to ensure that only one thread can increment the count at a time. 
- If we remove the `synchronized` keyword, both threads could increment the count simultaneously, leading to an incorrect final count due to race conditions.
- By synchronizing the `increment()` method, we ensure that the final count is always 2000, which is the expected result when both threads increment the count 1000 times each.


**Function to check if an array is a centered array:**
```java
import java.util.Scanner;
public class CenteredArray {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        ///  Enter the size of array
        System.out.println("Enter the size of the array:");
        int n = sc.nextInt();

        ///  Create array
        int[] array = new int[n];

        ///  Enter the array elements
        System.out.println("Enter the elements of the array:");
        for (int i = 0; i < n; i++){
            array[i] = sc.nextInt();
        }

        /// Check if the array is a centered array and print the result
        int result = centeredArray(array);

        ///  1= centered array, 0= not centered array
        if (result == 1) {
            System.out.println("The array is a Centered Array.");
        } else {
            System.out.println("The array is NOT a Centered Array.");
        }
    }

    /// Checks if the given array is a centered array, which means it contains all integers from 0 to n-1 exactly once and has an odd number of elements.
    public static int centeredArray(int[] array) {
        /// Check for null array and if array has odd number of elements
        if  (array == null || array.length % 2 == 0) {
            return 0;
        }

        int n = array.length;

        ///  Find the index of middle element
        int midindex = n / 2;

        ///  Find the value of middle element
        int midvalue = array[midindex];

        /// Check every elements
        for  (int i = 0; i < n; i++) {

            ///  Skip the middle element
            if (i == midindex) {
                continue;
            }
            ///  If the element is less than the middle value, return 0
            if (array[i] < midvalue){
                return 0;
            }
        }
        ///  If all elements are greater than middle elements
        return 1;
    }
}

```

## Question 2: How does JSP differ from Servlet and show the life cycle of Servlet? How do you create and read the cookies and session using JSP? Illustrate with an example.  

**Servlet vs JSP:**

| Aspect | JSP | Servlet |
|--------|-----|---------|
| Nature | JSP is a page-based technology | Servlet is a class-based technology |
| Usage | Used for designing UI (presentation layer) | Used for business logic (processing layer) |
| Coding Style | HTML with embedded Java code | Pure Java code |
| Compilation | JSP is compiled into Servlet internally | Servlet is already compiled Java class |
| Ease of Use | Easier for web designers | More suitable for developers |
| Performance | Slightly slower on first load | Faster execution after compilation |
| Maintenance | Easier to maintain for UI changes | Easier to maintain for business logic changes |
| File Extension | Uses .jsp extension | Uses .java extension |
| Execution | Executed on the server and generates HTML | Executed on the server and generates response |



**Servlet Life Cycle:**
- Servlet is a server-side Java program that handles client requests and generates dynamic web content. 
- The life cycle of a Servlet consists of the following stages:

  1. **Loading and Instantiation**
     - The servlet container loads the servlet class into memory when the server starts or when the first request is received.  
     - The class is loaded using the class loader of the container.  
     - After loading, an instance (object) of the servlet is created.  
     - This process happens only once during the life cycle of the servlet.  
     - At this stage, the servlet is not yet ready to handle client requests.  

  2. **init() Method**
     - After the servlet object is created, the container calls the `init()` method.  
     - This method is used to initialize the servlet and allocate resources.  
     - Resources such as database connections or configuration parameters are set up here.  
     - The `init()` method is executed only once in the servlet life cycle.  
     - If initialization fails, the servlet is not placed into service.  

  3. **service() Method**
     - Once initialized, the servlet is ready to handle client requests.  
     - For every client request, the container calls the `service()` method.  
     - The `service()` method determines the type of request (GET, POST, etc.).  
     - It internally calls methods like `doGet()` or `doPost()` based on request type.  
     - This method can be called multiple times for multiple client requests.  
     - It is the most important phase where actual processing takes place.  

  4. **destroy() Method**
     - When the servlet is no longer needed, the container calls the `destroy()` method.  
     - This method is used to release resources such as database connections or files.  
     - It is executed only once before the servlet is removed from memory.  
     - After this method is called, the servlet object becomes eligible for garbage collection.  
     - It marks the end of the servlet life cycle.  


**Creating and Reading Cookies using JSP:**

- **Creating a Cookie:**
    - A cookie is created usig the `Cookie` class in Java.
    - It stores small pieces of data on client browser
    - The cookie is added to the response object to be sent to the client.
  
  ```jsp
  <%
      // Create a cookie named "username" with value "JohnDoe"
      Cookie cookie = new Cookie("username", "JohnDoe");
      // Set the maximum age of the cookie to 7 days (in seconds)
      cookie.setMaxAge(7 * 24 * 60 * 60);
      // Add the cookie to the response
      response.addCookie(cookie);
  %>
  ```

- **Reading a Cookie:**
    - To read cookies, we retrieve the array of cookies from the request object.
    - We can then loop through the cookies to find the one we are interested in.

  ```jsp
    <%
        // Get the array of cookies from the request
        Cookie[] cookies = request.getCookies();
        String username = null;
    
        // Check if cookies are not null
        if (cookies != null) {
            // Loop through the cookies to find the "username" cookie
            for (Cookie cookie : cookies) {
                if (cookie.getName().equals("username")) {
                    username = cookie.getValue();
                    break;
                }
            }
        }
    %>
    ```


**Creating and Reading Session using JSP:**

- **Creating a Session:**
    - A session is created using the `HttpSession` object.
    - It allows you to store user-specific data across multiple requests.

  ```jsp
  <%
      // Get the current session or create one if it doesn't exist
      HttpSession session = request.getSession();
      // Set an attribute in the session
      session.setAttribute("user", "JohnDoe");
  %>
  ```

- **Reading a Session:**
    - To read session attributes, we retrieve the session object and then get the attribute value.

  ```jsp
    <%
        // Get the current session
        HttpSession session = request.getSession(false); // false means do not create a new session if it doesn't exist
        String user = null;
        
        // Check if session exists
        if (session != null) {
            // Get the "user" attribute from the session
            user = (String) session.getAttribute("user");
        }
    %>
    ```

**Example:**
```jsp

    // Create a cookie
```

## Question 8: What do you mean by JSP implicit objects? Discuss Java Mail API.  

**JSP Implicit Objects:**
- JSP implicit objects are predefined variables that are automatically available in JSP pages without the need for explicit declaration.
- These objects are created by JSP container to simplify web development.
- They are used to handle request, response, session and application data in JSP pages.
- They reduce the need for boilerplate code and allow developers to focus on the business logic.
- The commonly used JSP implicit objects include:
  - `request`: Used to access client request data such as parameters and headers.
  - `response`: Used to generate the response sent back to the client.
  - `session`: Used to store and retrieve user-specific data across multiple requests.
  - `application`: Used to store data that is shared across the entire application.
  - `out`: Used to send output to the client.
  - `config`: Used to access servlet configuration information.
  - `pageContext`: Used to access various objects and attributes related to the JSP page.
  - `page`: Refers to the current JSP page itself.
  - `exception`: Used to handle exceptions in error pages.
  
**Java Mail API:**
- Java Mail API is a library used to send and receive email messages in Java applications.
- It provides a platform-independent and protocol-independent framework for email communication.
- The Java Mail API supports various email protocols such as ***SMTP (Simple Mail Transfer Protocol)*** for sending emails, and ***POP3 (Post Office Protocol 3)*** and ***IMAP (Internet Message Access Protocol)*** for receiving emails.
- It allows developers to create email clients, send notifications, and manage email messages programmatically.

**Sending an Email using Java Mail API:**
1. Set up the mail server properties (e.g., SMTP host, port, authentication).
2. Create a mail session with the specified properties and authentication credentials.
3. Create a message object and set the sender, recipient, subject, and content of the email.
4. Use the `Transport` class to send the email message.
   
```java
import jakarta.mail.*;
import jakarta.mail.internet.*;
import java.util.Properties;

public class SendEmail {
    public static void main(String[] args) {

        // Email credentials
        final String username = "your_email@gmail.com";
        final String password = "app_password";

        // 1. Set up mail server properties    
        Properties props = new Properties();
        props.put("mail.smtp.host", "smtp.gmail.com");
        props.put("mail.smtp.port", "587");
        props.put("mail.smtp.auth", "true");
        props.put("mail.smtp.starttls.enable", "true");

        // 2. Create a mail session with authentication
        Session session = Session.getInstance(props,
            new Authenticator() {
                protected PasswordAuthentication getPasswordAuthentication() {
                    return new PasswordAuthentication(username, password);
                }
            });

        try {

            // 3. Create a message object and set email details
            Message message = new MimeMessage(session);
            message.setFrom(new InternetAddress(username));
            message.setRecipients(
                Message.RecipientType.TO,
                InternetAddress.parse("receiver@gmail.com")
            );
            message.setSubject("Java Mail API");
            message.setText("Hello from Java Mail API!");

            // 4. Send the email message
            Transport.send(message);
            System.out.println("Email Sent Successfully!");

        } catch (MessagingException e) {
            e.printStackTrace();
        }
    }
}
```

## Question 10: Write a TCP client-server system in which the client program sends two integers to a server program, which returns the greatest among them. 

**TCP Server:**
```java
import java.io.*;
import java.net.*;
public class TCPServer {
    public static void main(String[] args) {
        int port = 12345; // Port number to listen on

        try {
            // Step 2: Create a ServerSocket to listen for incoming connections
            ServerSocket serverSocket = new ServerSocket(port);
            System.out.println("Server is waiting");
            
            // Step 3: Accept a client connection
            Socket socket = serverSocket.accept();
            System.out.println("Client connected");

            // Step 4: Obtain input and output streams from the socket
            DataInputStream input = new DataInputStream(socket.getInputStream());
            DataOutputStream output = new DataOutputStream(socket.getOutputStream());

            // Step 5: Read two integers from the client
            int num1 = input.readInt();
            int num2 = input.readInt();

            // Step 6: Compute the greatest of the two integers
            int greatest = (num1 > num2) ? num1 : num2;

            // Step 7: Send the greatest integer back to the client
            output.writeInt(greatest);

            // Step 8: Close the connection
            socket.close();
            serverSocket.close();
        } catch (IOException e) {
            System.out.println(e);
        }
    }
}
```

**TCP Client:**
```java
import java.io.*;
import java.net.*;

public class TCPClient {
    public static void main(String[] args) {
        String host = "localhost"; // Server host
        int port = 12345; // Server port

        try {
            // Step 1: Create a Socket to connect to the server
            Socket socket = new Socket(host, port);

            // Step 2: Obtain input and output streams from the socket
            DataInputStream input = new DataInputStream(socket.getInputStream());
            DataOutputStream output = new DataOutputStream(socket.getOutputStream());

            // Step 3: Send two integers to the server
            output.writeInt(10);
            output.writeInt(20);

            // Step 4: Read the greatest integer from the server
            int greatest = input.readInt();
            System.out.println("Greatest integer received from server: " + greatest);

            // Step 5: Close the connection
            socket.close();

        } catch (IOException e) {
            System.out.println(e);
        }
    }
}
```

## Question 12: Write a program to create a menu named “File” with menu items “New,” “Save,” and “Exit”. 
```java
import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class MenuExample extends JFrame implements ActionListener {

    // Declare menu items
    JMenuItem newItem, saveItem, exitItem;

    public MenuExample() {
        // Set the title of window
        setTitle("Menu Example");

        // Set the size of the frame
        setSize(400, 300);

        // Create MenuBar
        JMenuBar mb = new JMenuBar();

        // Create Menu
        JMenu fileMenu = new JMenu("File");

        // Create Menu Items
        newItem = new JMenuItem("New");
        saveItem = new JMenuItem("Save");
        exitItem = new JMenuItem("Exit");

        // Add items to menu
        fileMenu.add(newItem);
        fileMenu.add(saveItem);
        fileMenu.add(exitItem);

        // Add menu to menubar
        mb.add(fileMenu);

        // Set menubar to frame
        setJMenuBar(mb);

        // Event Handling
        newItem.addActionListener(this);
        saveItem.addActionListener(this);
        exitItem.addActionListener(this);

        // Set frame visibility
        setVisible(true);
    }

    // Handle menu item actions
    public void actionPerformed(ActionEvent e) {
        if (e.getSource() == newItem) {
            System.out.println("New selected");
        } else if (e.getSource() == saveItem) {
            System.out.println("Save selected");
        } else if (e.getSource() == exitItem) {
            System.exit(0);
        }
    }

    public static void main(String[] args) {
        new MenuExample();
    }
}
```
