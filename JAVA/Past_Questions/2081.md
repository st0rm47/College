## Question 1: What is the task of the Listener interface? Write a socket program for a file server that makes a collection of files available for transmission to clients. When a client connects to the server, the server first reads a one-line command from the client. The command string can be of the form “GET <filename>”, where <filename> is a file name. The server checks whether the requested file actually exists. If so, it first sends the word “OK” as a message to the client. Then it sends the contents of the file and closes the connection. Otherwise, it sends “ERROR” to the client as message and closes the connection. Assume that there is no sub directories . 

**Listener Interface:**
- The Listener interface in Java is used in event-driven programming to handle events generated by GUI components. 
- It defines methods that are automatically called when a specific event occurs, such as a button click or mouse action.

**Task of the Listener Interface:**
- The Listener interface is used to receive and handle events in Java GUI applications.  
- It defines methods that respond to user actions like clicking, typing, or selecting.  
- It connects event sources (like buttons) with event handling code.  
- It allows applications to become interactive and responsive.  
- It works based on the event delegation model.  

**Listener Interface Works**
- An event source (e.g., JButton) generates an event.  
- A listener is registered to that component.  
- When an event occurs, the corresponding listener method is called automatically.  
- The event is processed inside that method.  

**Socket Program for File Server:**
```java
import java.io.*;
import java.net.*;
import java.util.Scanner;

public class FileServer {
    private static final int PORT = 12345;

    public static void main(String[] args) {
        try {

            // Create a server socket to listen for incoming connections
            ServerSocket serverSocket = new ServerSocket(PORT);
            System.out.println("File server is running on port " + PORT);

            // Wait for a client to connect
            Socket clientSocket = serverSocket.accept();
            System.out.println("Client connected");

            // Input stream to read data from the client
            BufferedReader input = new BufferedReader(new InputStreamReader(clientSocket.getInputStream()));

            // Output stream to send data to the client
            PrintWriter output = new PrintWriter(clientSocket.getOutputStream(), true);

            // Read the command from the client
            String command = input.readLine();

            // Split the command into parts to extract the filename
            String []parts = command.split(" ");

            // Check if the command is in the correct format and process it
            if (parts.length == 2 && parts[0].equals("GET"))
            {
                // Extract the filename from the command
                String filename = parts[1];

                // Create a File object for the requested filename
                File file = new File(filename);

                // Check if the file exists and is not a directory
                if (file.exists() && !file.isDirectory()) {

                    // Send "OK" to the client to indicate that the file is available
                    output.println("OK");

                    // Create a Scanner to read the contents of the file and send it to the client
                    Scanner fileScanner = new Scanner(file);
                    while (fileScanner.hasNextLine()) {
                        output.println(fileScanner.nextLine());
                    }
                    fileScanner.close();

                } else {
                    output.println("ERROR");
                }
            } else {
                output.println("ERROR");
            }

            clientSocket.close();
            serverSocket.close();
        } catch (IOException e) {
            System.out.println(e);
        }
    }
}

```

**Client Program:**
```java
package org.example.Midterm;

import java.io.*;
import java.net.*;

public class FileClient {
    private static final String SERVER_ADDRESS = "localhost";
    private static final int PORT = 12345;

    public static void main(String[] args) {
        try {

            // Create a socket to connect to the file server
            Socket serverSocket = new Socket(SERVER_ADDRESS, PORT);
            System.out.println("Connected to file server");

            // Input stream to receive data from the server
            BufferedReader input = new BufferedReader(new InputStreamReader(serverSocket.getInputStream()));

            // Output Stream to send data to the server
            PrintWriter output = new PrintWriter(serverSocket.getOutputStream(), true);

            // Read from keyboard
            BufferedReader userinput = new BufferedReader(new InputStreamReader(System.in));

            // Read the filename from the user
            System.out.println("Enter the filename to retrieve:");
            String filename = userinput.readLine();

            // Send the GET command to the server with the specified filename
            output.println("GET " + filename);

            // Read the response from the server
            String response = input.readLine();

            // If the response is "OK", read and print the contents of the file
            if (response.equals("OK")) {
                System.out.println("File contents:");

                // Read file content line by line
                String line;
                while ((line = input.readLine()) != null) {
                    System.out.println(line);
                }
            } else {
                System.out.println("File not found on server.");
            }

            serverSocket.close();
        } catch (IOException e) {
            System.out.println(e);
        }
    }
}


```
  
- In this implementation, the `FileServer` class listens for incoming client connections and processes commands to retrieve files.
-  The `FileClient` class connects to the server, sends a request for a file, and displays the contents if the file is found. 
-  The server checks for the existence of the requested file and responds accordingly.
-  This simple client-server application demonstrates how to use sockets for file transmission in Java.

## Question 6:  How do you write an IDL file and generate the stub and skeleton code for CORBA implementation? Explain. 

**IDL File:**
1. An IDL (Interface Definition Language) file is written using CORBA's IDL syntax.
2. It defines the interfaces, data types, and exceptions that will be used in the CORBA application.
3.  The IDL file typically has a .idl extension.
    ```idl
    module MyModule {
        interface MyInterface {
            string sayHello(in string name);
        };
    };
    ```

**Generating Stub and Skeleton Code:**
1. After writing the IDL file, you use an IDL compiler to generate the stub and skeleton code.
2. The following command is used to generate the code:
    ```bash
    idlj -fall MyModule.idl
    ```
3. This command generates required files for both client and server.
4. Stubs are generated for the client side.
5. Skeletons are generated for the server side. 

**Generated Files:**

The generated files include:
   - `MyInterface.java`: Interface that defines the methods declared in the IDL file.
   - `MyInterfaceHelper.java`: Helper class for communication and data type handling.
   - `MyInterfaceHolder.java`: Holder class for the interface.
   - `MyInterfaceStub.java`: Client-side stub for invoking methods on the server.
   - `MyInterfaceSkeleton.java`: Server-side skeleton for implementing the defined methods.
  
**Implementation:**
1. The server will implement the interface defined in the IDL file using the generated skeleton code.
2. The client will use the generated stub code to invoke methods on the server.
3. Both the client and server will need to initialize the ORB (Object Request Broker) and register the server object with the ORB for communication.
4. The client will then be able to call the methods defined in the IDL file, and the server will respond accordingly.
5. This process allows for seamless communication between the client and server in a CORBA-based application.

## Question 12: How do you insert pop-up menu? Distinguish between GET and POST request.  

**Inserting Pop-up Menu:**
1. In Java, you can insert a pop-up menu using the `JPopupMenu` class from the Swing library.
2. You can create a pop-up menu and add menu items to it.
3. Then, you can show the pop-up menu at a specific location when a certain event occurs (e.g., right-click).

```java
import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class PopupMenu extends JFrame {
    public PopupMenu() {

        // Set the title of the frame
        setTitle("Popup Menu");

        // Set the size of the frame
        setSize(300, 200);

        // Create a pop-up menu
        JPopupMenu pm = new JPopupMenu();

        // Create menu items
        JMenuItem cut = new JMenuItem("Cut");
        JMenuItem copy = new JMenuItem("Copy");

        // Add menu items to the pop-up menu
        pm.add(cut);
        pm.add(copy);

        // Add mouse listener to the frame to show the pop-up menu
        addMouseListener(new MouseAdapter() {
            public void mousePressed(MouseEvent e) {
                if (e.isPopupTrigger())
                    pm.show(e.getComponent(), e.getX(), e.getY());
            }

            public void mouseReleased(MouseEvent e) {
                if (e.isPopupTrigger())
                    pm.show(e.getComponent(), e.getX(), e.getY());
            }
        });

        // Set the visbility
        setVisible(true);

    }

    public static void main(String[] args) {
        new PopupMenu();
    }
}

```

**Distinction between GET and POST request:**
|Aspect|GET Request|POST Request|
|---|---|---|
|Purpose|Used to request data from a specified resource.|Used to submit data to be processed to a specified resource.|
|Data Transmission|Data is sent in the URL as query parameters.|Data is sent in the body of the request.|
|Security|Less secure as data is visible in the URL.|More secure as data is not visible in the URL.|
|Data Length|Limited by the URL length (usually around 2048 characters).|No size limitations on the data being sent.|
|Caching|GET requests can be cached by browsers and servers.|POST requests are not cached.|
|Use Cases|Used for retrieving data, such as fetching a webpage or an API response.|Used for sending data, such as submitting a form or uploading a file.|
