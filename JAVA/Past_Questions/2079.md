## Question 4:  What is multithreading? How can you create multithreaded program in Java? Explain.  

**Multithreading**
- Multithreading is the process of executing multiple threads at the same time within a program.  
- A thread is a lightweight unit of a process.  
- It allows parallel execution of tasks.  
- It is widely used in modern applications such as GUI, web servers, and games. 
- It can improve the performance of a program by utilizing the CPU efficiently. 

**Creating a Multithreaded Program in Java**
- In Java, you can create a multithreaded program by either extending the `Thread` class or implementing the `Runnable` interface.

1. **Extending the Thread Class**:
   - You can create a new class that extends the `Thread` class
   - The `run()` method is overridden to define the code that will be executed by the thread. 
   - Then, you can create an instance of this class and call the `start()` method to begin execution.

    ```java
    class MyThread extends Thread {
        public void run() {
            System.out.println("Thread is running");
        }
    }
    public class MultithreadingExample {
        public static void main(String[] args) {
            MyThread thread1 = new MyThread();
            thread1.start();
        }
    }
    ```

2. **Implementing the Runnable Interface**:
   - You can create a class that implements the `Runnable` interface
   - The `run()` method is implemented to define the code that will be executed by the thread. 
   - Then, you can create an instance of the class  and pass it to a `Thread` object, and call the `start()` method.

    ```java
    class MyRunnable implements Runnable {
        public void run() {
            System.out.println("Thread is running");
        }
    }
    public class MultithreadingExample {
        public static void main(String[] args) {
            MyRunnable myRunnable = new MyRunnable();
            Thread thread1 = new Thread(myRunnable);
            thread1.start();
        }
    }
    ```

## Question 5: What is grid layout? Compare grid layout with grid bag layout.

**Grid Layout**
- GridLayout is a layout manager in Java AWT used to arrange components in a grid structure.  
- It divides the container into equal-sized rows and columns.  
- Each component occupies exactly one cell in the grid.  
- Components are added in row-wise order automatically.  
- It is useful for designing calculators, forms, and button panels.  

```java
import java.awt.*;
import javax.swing.*;
public class GridExample extends JFrame {
    public GridExample() {
        // Set the title of the frame
        setTitle("Grid Layout Example");

        // Set the size of the frame
        setSize(300, 200);

        // Set the layout manager to GridLayout with 2 rows and 3 columns
        setLayout(new GridLayout(2, 3));

        // Add buttons to the frame
        add(new JButton("1"));
        add(new JButton("2"));
        add(new JButton("3"));
        add(new JButton("4"));
        add(new JButton("5"));
        add(new JButton("6"));

        // Set the visibility of the frame
        setVisible(true);
    }

    public static void main(String[] args) {
        new GridExample();
    }
}
```

**Grid Layout vs GridBagLayout**
| Feature               | GridLayout                          | GridBagLayout                         |
|-----------------------|-------------------------------------|---------------------------------------|
| Layout Structure      | Divides container into equal cells   | Flexible grid with varying cell sizes  |
| Component Placement   | Components are placed in order       | Components can be placed anywhere      |
| Component Size        | All components are the same size      | Components can have different sizes     |
| Alignment             | Components are centered in cells      | Components can be aligned as needed     |
| Complexity            | Simple to use                        | More complex to use                    |
| Control                | Limited control over component placement | High control over component placement    |
| Use Case              | Suitable for simple layouts           | Suitable for complex layouts            |
| Performance            | Faster for simple layouts             | Slower due to flexibility              |
| Example Usage         | Calculator buttons, forms             | Complex forms, dashboards              |
